SageMaker Inference Mode Decision

Match inference mode to latency and traffic requirements.

Real-Time Endpoint
Persistent, low-latency synchronous API. Pay per hour. Use when SLA requires < 1s response.
Serverless Inference
Scales to zero, pay per invocation. Use for sporadic/unpredictable traffic with no idle cost requirement.
Async Inference
Background processing, result in S3. Use for large payloads (>6 MB) or long jobs (>60s).
Batch Transform
Offline bulk scoring from S3. Use for scheduled batch inference on millions of records.

SageMaker Model Monitor — Four Types

Data Quality Monitor

Detects input feature drift: null rates, type mismatches, value range violations. No ground truth needed.

Fix: Investigate upstream data pipeline for schema changes.
Model Quality Monitor

Tracks accuracy/error rate vs ground truth labels over time. Requires ground truth ingestion.

Fix: Trigger retraining pipeline when metrics fall below threshold.
Bias Drift Monitor

Detects fairness metric drift (Disparate Impact, Equal Opportunity) across demographic groups.

Fix: Retrain with updated fairness constraints; audit data collection.
Feature Attribution Drift

Detects SHAP value distribution changes — model relying on different features than at training.

Fix: Investigate data quality; consider retraining.

Distributed Training Strategy

Choose based on model size vs. data size.

SageMaker Distributed Data Parallel (SDP)

Model fits on one GPU. Data split across workers. AllReduce gradient aggregation.

Use when: Standard deep learning models (ResNet, BERT) on large datasets.
SageMaker Distributed Model Parallel (SMP)

Model exceeds single GPU memory. Layers split across GPUs via pipeline/tensor parallelism.

Use when: LLMs, GPT-scale models that don't fit on a single GPU.
